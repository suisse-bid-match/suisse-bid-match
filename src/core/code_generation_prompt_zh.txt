我想做一个一个面向灯具投标场景的“端到端匹配”项目，具体匹配投标中所需求的灯具产品到供应商公司所能提供的灯具产品，通过比较灯具技术参数来匹配；
实现主要依赖LLM(OpenAI API)的能力，是一个多步骤请求LLM的过程；
在这个过程中，我会提供一些输入给LLM，LLM会根据这些输入生成一些输出，这些输出会被我用来进行下一步的请求；

OpenAI API的key（用来后续请求LLM）和模型选择、
供应商公司产品数据库的配置信息（主要关注vw_bid_products、vw_bid_specs，包含了所有技术参数）
都可以在.env中找到



多步骤请求的过程如下(里面包含了LLM和非LLM的步骤)：
{

        "output": "提取出多个灯具在投标中所需求的相关参数信息，如型号、数量、技术参数等，只抽取参数名和参数值（或者范围），其中参数名需要和数据库schema对齐（写成table.field的形式，方便后续直接在此基础上构建SQL语句），需要输出成json"，

        "output": "决定投标过程中的软硬约束，哪些参数（字段）是一定要满足的，哪些允许一点点的不满足，通过设置一个is_hard来表明是否是硬约束，尽量将要求设置宽松一点，只把最重要的设置成硬约束，因为后面我们需要用硬约束写SQL查询数据库，我们目的是过滤一部分数据，而不是直接选出那个产品，硬约束少一点有助于我们找到更多的候选产品，但是同时也要保证真正的硬约束不能被忽略"

    "step5": {
        "input": "step4的输出"，

        "note": "不需要llm，这步只需要json操作、字符串匹配，SQL模板建立就可以完成"

        "note": "不需要llm"
